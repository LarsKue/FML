2 Asymptotic Error of the NN classifier for the Toy Example¶

We recall the toy example, where the feature variable $x\in \left[0 , 1\right]$ is real-valued and 1-dimensional and the response $y\in \left\{0 , 1\right\}$ is discrete with two classes. The prior probabilities and likelihoods are given by

$$p(y = 0) = p(y = 1) =1/2$$$$p(x\,|\,y = 0) = 2 - 2x$$$$p(x\,|\,y = 1) = 2x$$

We can easily compute the density of $x$ by marginalization over $y$

$$p(x) = p(x\,|\,y = 0)\,p(y = 0)+p(x\,|\,y = 1)\,p(y = 1) = (2 - 2x)\cdot 1/2 + 2x\cdot1/2 = 1$$

The posteriors are

$$p^*(y=0\,|\,x) = \frac{p(x\,|\,y = 0)\,p(y = 0)}{p(x)} = 1-x $$$$p^*(y=1\,|\,x) = \frac{p(x\,|\,y = 1)\,p(y = 1)}{p(x)} = x$$

The nearest-neighbor classiffier's asymptotic local error is given by the Gini impurity

$$p_{\infty}(\text{error}\,|\,x) = 1 - p^*(y=0\,|\,x)^2 - p^*(y=1\,|\,x)^2 = 1- (1 - x)^2 -x^2 = 2(x - x^2)$$

The asymptotic total error for the nearest neighbor classiffier for the toy example is the expectation of the local error over all $x$

$$p_{\infty}(\text{error}) = \int_0^1 p_{\infty}(\text{error}\,|\,x)\,p(x)\,dx = \int_0^1 2(x - x^2)\,dx= \frac{1}{3} = 33.33\%$$

You can use this theoretical result to interpret your experimental findings in exercise 1a. There the computed average error of the NN classifier with training sets of size 2 was 35%, i.e. pretty close to 33.33%. With training sets of size 100, it got even closer (almost exactly 33.33%) with lower standard deviation. An asymptotic total error of 33.33% suggests that the average error rate will converge against this value if we would calculate the average over the whole population.

3.3 Distance function computation using vectorization¶

The Euclidean distance between vectors $\textbf{a}, \textbf{b} \in \mathbb{R}^N$ is

\begin{align} \text{dist}(\textbf{a}, \textbf{b}) &= \sqrt{ (a_1 - b_1)^2 + (a_2 - b_2)^2 + \ldots }. \end{align}

In order to compute this for all $\textbf{a}$ that are rows of $\textbf{A}=$ X_train $\in \mathbb{R}^{I \times N}$ and all $\textbf{b}$ that are rows of $\textbf{B}=$ X_test $\in \mathbb{R}^{J \times N}$, we can contruct a big matrix $C \in \mathbb{R}^{I \times J \times N}$ containing all the differences $A_{i,n} - B_{j,n}$.

numpy has a useful feature called broadcasting which helps us here. By adding a dimension to $\textbf{A}$ to be in $\mathbb{R}^{I \times 1 \times N}$, and $\textbf{B}$ to be in $\mathbb{R}^{1 \times J \times N}$ (both with np.expand_dims()), the matrices will automatically be replicated along the $1$-dimension to match each other's shapes when we subtract.

The resulting $I \times J \times N$ matrix can then be squared element-wise, summed up along the $N$-dimension, and the square root taken element-wise, which leaves us with the desired $I \times J$ distance matrix.

Alternative vectorized approach¶

Alternatively, the Euclidean distance between vectors $\textbf{a}, \textbf{b} \in \mathbb{R}^N$ can be transformed as follows:

\begin{align} \text{dist}(\textbf{a}, \textbf{b}) &= \sqrt{ (a_1 - b_1)^2 + (a_2 - b_2)^2 + \ldots } \\ &= \sqrt{ (a_1^2 + a_2^2 + \ldots) + (b_1^2 + b_2^2 + \ldots) - (2 a_1 b_1 + 2 a_2 b_2 + \ldots) } \\ &= \sqrt{ \sum_i a_i^2 + \sum_i b_i^2 - 2 \cdot \textbf{a}^\top \cdot \textbf{b} } \\ \end{align}

This computation can also be performed for many vectors at once by operating on matrices $\textbf{A} \in \mathbb{R}^{I \times N}$ and $\textbf{B} \in \mathbb{R}^{J \times N}$, which contain the $\textbf{a}$s and $\textbf{b}$s as rows. This method turns out to be considerably faster, since the largest constructed matrix is of size $I \times J$, as opposed to the $I \times J \times N$-matrix we constructed in the previous vectorized method.

In [10]:
def dist_vec_alternative(training, test):
    AB = np.dot(test, np.transpose(training)) # shape is (n_test, n_train)
    AA = np.sum(np.square(test), axis = 1) # shape is (n_test,)
    BB = np.sum(np.square(training), axis = 1) # shape is (n_train,)

    dist_matrix = np.sqrt(np.transpose(BB - 2 * AB) + AA) # shape is (n_train, n_test)

    return dist_matrix

# k-nearest-neighbor classifier based on vectorized distance computation
def k_nearest_neighbors(X_train, y_train, X_test, k):
    # find k nearest neighbors for each test image
    dist = dist_vec_alternative(X_train, X_test)
    indices = np.argsort(dist, axis = 0)[:k,:] # 2D array containing indices of k neighbors with lowest distance
    neighbors = y_train[indices] # indexing y_train (1D) with indices (2D) gives 2D array of neighbors' classes!

    # find most common class among neighbors of each test image
    # (you could also use np.bincount() or scipy.stats.mode() for the voting)
    counts = np.zeros((10, X_test.shape[0]))
    for i in range(k):
        for j in range(X_test.shape[0]):
            counts[neighbors[i,j], j] += 1

    return np.argmax(counts, axis = 0)

# use KNN classifier to distinguish two classes
def knn_compare(classA, classB, k):
    X_train_AB = X_train[np.logical_or(classA == y_train, classB == y_train)]
    y_train_AB = y_train[np.logical_or(classA == y_train, classB == y_train)]

    X_test_AB = X_test[np.logical_or(classA == y_test, classB == y_test)]
    y_test_AB = y_test[np.logical_or(classA == y_test, classB == y_test)]
    
    result = k_nearest_neighbors(X_train_AB, y_train_AB, X_test_AB, k)

    return 100 * (1 - np.count_nonzero(result == y_test_AB) / len(result))

Why no loops?¶

As our distance matrix example showed, loops are often very inefficient in a scripting language such as Python. The comparison and indexing functionality of numpy, however, uses highly optimized code and will generally have better performance. If you can express an operation in numpy-instructions without sacrificing too much readability, do it!

def cross_validate(n_folds, k = 1, shuffle = True, n_repetitions = 20):
    sklearn_knn = KNeighborsClassifier(n_neighbors = k)

    results = []
    for i in range(n_repetitions if shuffle else 1):
        error_rates_own = []; error_rates_sklearn = []

        # (shuffle and) split up the data
        if shuffle: perm = np.random.permutation(range(len(y_all)))
        else:       perm = range(len(y_all))
        X_folds = np.array_split(X_all[perm], n_folds)
        y_folds = np.array_split(y_all[perm], n_folds)

        for n in range(n_folds):
            # construct train and test set from folds
            X_train_f = np.concatenate([X_folds[i] for i in range(n_folds) if i != n])
            X_test_f = X_folds[n]
            y_train_f = np.concatenate([y_folds[i] for i in range(n_folds) if i != n])
            y_test_f = y_folds[n]

            # run our own nearest neighbor classifier
            result_own = k_nearest_neighbors(X_train_f, y_train_f, X_test_f, k)
            error_rates_own.append(100 * (1 - np.count_nonzero(result_own == y_test_f) / len(result_own)))

            # run sklearn's nearest neighbor classifier
            sklearn_knn.fit(X_train_f, y_train_f)
            result_sklearn = sklearn_knn.predict(X_test_f)
            error_rates_sklearn.append(100 * (1 - np.count_nonzero(result_sklearn == y_test_f) / len(result_sklearn)))

        results.append(np.array(
                [[np.mean(error_rates_own), np.std(error_rates_own)],
                 [np.mean(error_rates_sklearn), np.std(error_rates_sklearn)]]
            ))

    return np.mean(np.concatenate(results), axis = 0)

The two classifiers seem to yield identical results.

When using more folds, the training set gets larger. This causes the error rate to decrease.

But using more folds also means that we evaluate the models on a greater number of different (and smaller) test sets. This causes the variance to increase.

Shuffling the data prior to creating folds significantly decreases the average error in all configurations. This suggests that the data set is ordered in such a way that different sections have different characteristics. A likely explanation is that the digits might be arranged by author, and the NN classifier generalizes worse to handwriting styles not present in the training data.
